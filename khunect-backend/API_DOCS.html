💡 이후 모든 인증 API는 HTTP Header에 Authorization: Bearer {accessToken} 을 포함해야 합니다.
POST /api/auth/exchange 구글 OAuth2 code → JWT 교환 ▾
Request Body
필드타입필수설명
codeString필수Google OAuth2 인가 코드

"code": "4/0AX4XfWi..."
Response
{
  "accessToken":  "eyJhbGci...",
  "refreshToken": "eyJhbGci...",
  "signupCompleted": false,
  "user": {
    "id": 1,
    "email": "student@khu.ac.kr",
    "nickname": null
  }
}
⚠️ signupCompleted가 false이면 프론트에서 회원가입 추가정보 입력 화면으로 이동시켜야 합니다.
POST /api/auth/refresh 액세스 토큰 갱신 ▾
Request Body
필드타입필수설명
refreshTokenString필수리프레시 토큰
Request
"refreshToken": "eyJhbGci..."
Response
{
  "accessToken":  "eyJhbGci...",
  "refreshToken": "eyJhbGci..."
}
POST /api/auth/logout 로그아웃 🔑 Auth ▾

Match Post — 매칭 게시글

/api/match-posts
POST /api/match-posts 매칭 게시글 작성 🔑 Auth ▾
Request Body
필드타입필수조건
preferredTimeTextString필수최대 100자 (예: "평일 오후")
locationTextString필수최대 100자 (예: "중앙도서관")
contentString필수최대 1000자
categoryString선택최대 30자 (예: "스터디")
Request
{
  "preferredTimeText": "평일 오후 2시",
  "locationText":     "중앙도서관 3층",
  "content":          "알고리즘 같이 공부해요",
  "category":         "스터디"
}
Response (MatchPostResponse)
{
  "id":               42,
  "authorUserId":    1,
  "authorNickname":  "김경희",
  "preferredTimeText":"평일 오후 2시",
  "locationText":    "중앙도서관 3층",
  "content":         "알고리즘 같이 공부해요",
  "category":        "스터디",
  "status":          "OPEN",
  "acceptedByUserId": null,
  "acceptedAt":      null,
  "createdAt":       "2026-04-02T12:00:00"
}
GET /api/match-posts 매칭 게시글 목록 ▾
Query Parameters
파라미터타입기본값설명
statusString없음OPEN / MATCHED / CLOSED
pageint0페이지 번호
sizeint10페이지 크기
PATCH /api/match-posts/{id} 게시글 수정 🔑 Auth ▾
작성자 본인만 수정 가능. Request Body는 POST와 동일.
DELETE /api/match-posts/{id} 게시글 삭제 🔑 Auth ▾
작성자 본인만 삭제 가능.
POST /api/match-posts/{id}/accept 매칭 수락 → 1:1 채팅방 자동 생성 🔑 Auth ▾
Response
{
  "matchPostId":  42,
  "status":       "MATCHED",
  "directRoomId": 2   // 생성된 1:1 채팅방 ID → 바로 채팅으로 이동
}
💡 수락 즉시 작성자↔수락자 간 1:1 채팅방이 생성됩니다. directRoomId로 바로 채팅 화면으로 이동하세요.
